# Conventions

This file is included via `@include` in `SKILL.md` to demonstrate how skills can split large prompts across files.

## Output Format

- Use markdown headers for sections.
- Keep summaries concise (3-5 bullet points max).
- Always include file paths when referencing code.

## Code Style

- Follow the project's existing code style.
- Prefer editing existing files over creating new ones.
- Do not add unnecessary comments or docstrings.
